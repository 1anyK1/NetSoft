## Сводка REST vs gRPC

| Протокол | Наблюдение |
|----------|-------------|
| **REST** | больше накладные расходы JSON + HTTP-парсинга на каждый вызов |
| **gRPC** | компактнее тело сообщения и меньше копирований на стороне клиента |

## Server Streaming

В контракте `weeks/week-07/proto/service.proto` добавлен `Subscribe(...) returns (stream NotificationEvent)` и реализован в `grpc_server.py` простым генератором из нескольких событий.

## Вывод

Для **notifications-s01** Unary gRPC почти всегда выигрывает в чистых замерах «много маленьких вызовов», а REST остаётся проще для внешних интеграций и браузерных клиентов.
