# Архитектура (comments-s01)

Финальный проект (неделя 17, вариант **comments-s01**) состоит из двух микросервисов, уже реализованных на промежуточных неделях и спрятанных за общим **API Gateway**:

1. **comments-svc** — REST приложение недели **03** (`comments`, префикс `/api/comments`).
2. **items-graphql-svc** — приложение недели **05**, GraphQL на `/graphql` (bounded context каталога).
3. **gateway** — Nginx, единственная точка входа во внутреннюю Docker-сеть.

Внешний клиент может вызывать оба домена через один адрес `:9080`, не зная портов внутренних сервисов.

## Протоколы и обоснование

REST описывает стандартные CRUD операции над комментариями. GraphQL оставляет фронту свободу выбирать поля и уменьшает число связанных HTTP-ресурсов. Для синхронного межсервисного общения с низкой задержкой вариант недели можно расширить gRPC-слоем (**недели 07–08**), сохраняя HTTP на edge.

## Доставка

Каждый сервис упакован в Docker. Композ ниже добавляет второй домен без дублирования кода Helm/CI (существующие `tickets-app` chart и пайплайн `.github/workflows/ci.yml` переиспользуют общие практики репозитория).

## Локальный запуск одной командой

```bash
cd weeks/week-17
docker compose up --build
```

Комментарии:

```bash
curl http://127.0.0.1:9080/api/comments
```

GraphQL:

```bash
curl -s -X POST http://127.0.0.1:9080/graphql \
     -H "content-type: application/json" \
     -d '{"query":"query { items { id name } }"}'
```
