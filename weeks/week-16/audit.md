# Аудит безопасности (devices-s01)

Проект: **devices-s01** (`devices` ресурс, межсервисный трафик).

| Находка | Риск | Рекомендация |
|---------|------|--------------|
| In-memory-хранилище без ACL | высокий | Развести домены доверенных клиентов, добавить аутентификацию перед CRUD |
| Отсутствие rate-limit на публичном REST | средний | Nginx/Ingress `limit_req` или сервисный middleware |
| Секреты в CI/Git | критический | GitHub/GitLab секреты, не хардкодить токены в compose |
| Слабая наблюдаемость | средний | структурные логи + трассировка + алерты |
| Известные CVE base image | переменный | регулярный `docker scan`/`trivy` на образ |
| Replay API вызовов | средний | идемпотентные ключи на мутациях |
| Отсутствие WAF перед публичным ingress | информационный | включить CDN/WAF на edge |

## Компенсации

- Принять **Argon2/bcrypt** для пользовательских паролей если появится auth.
- Включить **TLS** между клиент ↔ gateway и между сервисами (mTLS в mesh).

## Комплексный статус по чеклисту

Пройдены пункты 1–3 вручную (код приложения недель 01–05): критических уязвимостей в простом каркасе не выявлено, однако боевые риски требуют вышеперечисленной инфраструктуры.
