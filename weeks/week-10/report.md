# Отчет по Docker (bookings-s01)

## Размер образа

Итоговый размер образа после `docker build` можно посмотреть командой `docker images`: колонка **SIZE** показывает сжатый размер.

## Слои (layers)

Multi-stage сборка даёт меньше слоёв в финальном образе: стадия **builder** не попадает в runtime, каждый `COPY` и `RUN` в финальной стадии добавляет отдельный **layer**.

## Команды

- Сборка: `docker build -t bookings-s01 -f Dockerfile .`
- Запуск: `docker run -p 8112:8112 bookings-s01`

## Сравнение «до/после»

Если временно собрать «одностадийный» Dockerfile с установкой всех зависимостей в финальном слое, столбец `SIZE` будет заметно больше такого же multi-stage варианта, потому что в образ не попадут промежуточные pip-кэши и временные сборочные утилиты.
